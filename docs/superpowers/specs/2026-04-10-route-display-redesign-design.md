# 路径显示重设计方案

## 背景

当前地面站已经能在 Qt6 地图上预绘制整条路径，并通过颜色、箭头和重复编号展示多次经过的航段。但从实际界面反馈来看，重复路径信息仍然不够直观，主要问题有三点：

1. 非重复路径与重复路径的视觉权重过于接近，用户需要主动辨认。
2. 重复路径同时使用平行偏移、不同颜色和数字编号，信息层次偏杂。
3. 在 9x7 的小尺度全局地图中，过多的局部细节会削弱整体可读性。

本次改版的目标是把重复路径从“需要分析才能看懂”改成“第一眼就能看见”。

## 目标

在不改变现有通信链路、路径规划器和实时红球移动逻辑的前提下，重构路径显示方式，使地图满足以下要求：

- 红球起飞前先完整显示整条航线。
- 普通航段保持低干扰、统一的主路径风格。
- 重复经过的航段使用单独高亮带强调。
- 重复航段使用统一高亮色，不再区分第 2 次、第 3 次的不同颜色。
- 重复航段中央显示一个常驻圆牌，标明总重复次数，例如 `2x`、`3x`。
- 方向信息继续由箭头表达，但箭头仅作为辅助，不再承担主要辨识职责。
- 起飞区、终点和降落走廊继续保留清晰标识。

## 已确认的方案

### 总体策略

采用“方案 A1 + 中央圆牌”的组合：

- 主航线：统一细蓝线。
- 重复航段：在主航线上叠加一条统一的橙色高亮带。
- 重复次数：在重复航段中部放置圆形徽标，文字为 `2x`、`3x` 等总次数。
- 方向：重复航段末端保留较大的方向箭头。

### 为什么放弃旧方案

旧方案把同一条边的多次经过拆成多个平行车道，并对每次经过分别编号。这种表达更像调试视图，适合算法开发，不适合比赛演示，因为：

- 地图尺度有限，平行车道会显得拥挤。
- 用户需要先理解“颜色代表第几次经过”，认知成本偏高。
- 重点应该是“这里发生了重复”，而不是“每一次重复的顺序细节”。

### 为什么选择中央圆牌

重复航段如果只高亮不写次数，虽然最干净，但无法让评委快速判断重复程度。  
中央圆牌方案在“醒目”和“简洁”之间更平衡：

- 一眼看出该边是重复航段。
- 同时直接知道重复总次数。
- 不需要额外查图例，也不需要理解多层颜色编码。

## 视觉规则

### 1. 普通航段

- 统一使用单条细蓝线。
- 线宽保持较轻，作为全局路径骨架。
- 箭头可以保留，但视觉权重低于重复航段。

### 2. 重复航段

- 在普通蓝线之上叠加一条更粗的橙色高亮带。
- 不再使用平行偏移车道来区分每次经过。
- 同一条边无论重复 2 次还是 3 次，都只显示为一条高亮带。

### 3. 次数圆牌

- 放置在重复航段中部，优先居中。
- 文字统一为 `Nx` 格式，例如 `2x`、`3x`。
- 圆牌背景与重复航段高亮色一致，文字为白色粗体。
- 圆牌尺寸需要在远看时仍可辨认，但不能过大遮挡网格标签。

### 4. 箭头

- 普通航段箭头保持小尺寸。
- 重复航段箭头适当放大，与橙色高亮带统一风格。
- 箭头只负责辅助说明方向，不再依赖多箭头多颜色来解释重复次序。

### 5. 起点与终点

- 起飞区继续使用蓝色标识。
- 当前无人机继续使用红球显示。
- 终点和降落走廊继续使用绿色语义。

## 组件职责

### `RouteVisualizer`

负责从原始路线中计算“边级统计信息”，但输出语义要从“每次经过的车道”转向“边是否重复、总重复次数是多少”。

建议输出的核心信息包括：

- `from_cell`
- `to_cell`
- `is_repeated`
- `repeat_count`
- `direction`

如果实现时为了兼容已有代码继续保留 `pass_index` 等字段，也应保证 UI 层不再依赖它们做平行偏移显示。

### `GridScene`

负责最终绘制：

- 绘制基础主航线。
- 对重复边叠加高亮带。
- 放置 `2x / 3x` 圆牌。
- 控制普通箭头与重复箭头的视觉层级。
- 保持起飞区、红球、终点、降落走廊的现有语义。

### `MainWindow`

负责图例文案同步更新：

- 删除“首次经过 / 第二次经过 / 第三次及以上”这种旧语义。
- 改成“蓝线为主航线，橙色高亮表示重复航段，圆牌表示重复次数”。

## 错误处理与兼容性

- 如果收到非法路径，仍沿用当前告警机制输出 warning，不因为 UI 重构改变校验逻辑。
- 如果某条边重复次数为 1，则绝不显示高亮带和次数圆牌。
- 如果某条边长度异常或无法绘制箭头，应至少保证主线存在，不因装饰层失败而丢失路径主体。
- 如果圆牌与终点或起点标签过近，应优先保证终点/起飞区标识不被遮挡。

## 测试要求

至少覆盖以下验证点：

1. 重复边识别仍然基于无向边统计。
2. 当某条边重复两次或以上时，能得到正确的总重复次数。
3. 非重复边不会被标记为重复。
4. 图例文案与新显示语义一致。

## 不在本轮范围内

本轮不做以下内容：

- 路径显现动画。
- 缩放级别自适应文字。
- 鼠标悬停弹层细化。
- 算法或通信协议变更。

## 实施原则

- 优先小步修改现有 `RouteVisualizer` 和 `GridScene`，避免大规模重构。
- 不破坏现有通过的通信联调和数据库记录能力。
- 保持 Qt6 Widgets 现有架构不变，仅调整显示层语义与样式。
