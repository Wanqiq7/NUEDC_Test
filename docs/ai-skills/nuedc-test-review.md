# nuedc-test-review

## 触发场景

当任务完成后准备提交、合并或交付联调前使用。

## 审查重点

- 通用层是否仍然独立于题目层。
- 新题目是否只通过 Adapter/Runtime/Codec 接入。
- 通信失败是否有测试和可读错误。
- 规划算法是否覆盖边界案例和成本回归。
- UI 状态是否避免误操作。

## 禁止事项

- 不只跑单个快乐路径测试就声称完成。
- 不忽略构建失败、跳过测试或测试环境缺失。
- 不把生成文件当作手工源码审查重点。

## 必跑测试

```bash
cmake --build build
ctest --test-dir build --output-on-failure
```

## 推荐提示词

请按架构边界、通信可靠性、规划边界、UI 状态和测试覆盖五个维度审查当前 diff，优先列出会影响比赛现场的风险。
